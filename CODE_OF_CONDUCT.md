# Code of conduct

Be decent. Disagree about code; don't attack the people writing it.

## In bounds

Strong technical opinions, vigorous pushback on a design, rejecting
a contribution, asking for changes — all fine. anchord has explicit
anti-patterns (see [CONTEXT.md](CONTEXT.md)) and PRs that hit them
get pushed back, sometimes hard. That's the work, not bad behavior.

## Out of bounds

- Personal attacks, name-calling, sustained hostility.
- Harassment, deliberate misgendering, slurs of any kind.
- Sharing someone's personal information without consent.
- Sexual or violent content in any project space.

## Reporting

Use GitHub's private vulnerability reporting flow — it doubles as a
private channel for code-of-conduct issues:

→ https://github.com/AlexCherrypi/anchord/security/advisories/new

Mention "code of conduct" in the title so it routes correctly.

The maintainer will review every report. Outcomes range from
"misunderstanding cleared up, both sides apologized, done" to
permanent project ban; the response depends on the situation.

## Inspiration

The text is original but the underlying expectations align with the
[Contributor Covenant](https://www.contributor-covenant.org/) — if
that's the document you're more familiar with, the substance is
the same.
